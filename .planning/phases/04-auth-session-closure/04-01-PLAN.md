| Boundary | Description |
|----------|-------------|
| browser storage → auth bootstrap | Untrusted local/session storage influences session restoration and redirect intent |
| browser shell → Convex auth actions | Public client requests can trigger sign-in, refresh, and sign-out transitions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | S | `src/lib/auth/auth0.ts` | mitigate | Accept redirect intent only from app-local protected paths and clear malformed intent before restore so auth entry cannot spoof arbitrary destinations. |
| T-04-02 | T | browser token storage + `fetchAccessToken` | mitigate | Centralize token-clear + `authState` transition logic so refresh failure cannot tamper the UI into showing a false signed-in state. |
| T-04-03 | I | protected-route restore messaging | accept | Intent labels expose only route purpose (`scanner`, `that book`) and not sensitive record data; acceptable low-risk disclosure. |
| T-04-04 | D | auth action queue | mitigate | Keep serialized auth actions and add regression coverage for repeated sign-in/reload/sign-out transitions so overlapping auth requests do not strand the shell. |

</threat_model>
