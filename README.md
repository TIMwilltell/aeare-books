# AeAre Books

A progressive web app for parents to scan book barcodes, auto-populate metadata (title, author, ISBN) from Open Library, attempt AR level/points lookup from arbookfind.com, and track reading progress and AR quiz scores for children.

## Features

- **Barcode Scanning** — Scan book barcodes with your phone's camera to quickly add books
- **Auto-Populate Book Data** — Automatically fetch title, author, and cover image from Open Library
- **AR Level Integration** — Fetch Accelerated Reader (AR) levels from arbookfind.com (with manual fallback)
- **Reading Progress Tracking** — Track when books are read with date tracking
- **Quiz Score Recording** — Record AR quiz scores for each book
- **Graceful Fallbacks** — If AR lookup fails, manual entry keeps the workflow moving
- **PWA** — Installable to home screen, works on iOS and Android

## Tech Stack

- **Frontend:** SvelteKit
- **Backend:** Convex (real-time database & serverless functions)
- **Local Storage:** Client-side persistence is partial and under active verification
- **Barcode Scanning:** Quagga
- **APIs:** Open Library, arbookfind.com (scraped)
- **Build:** Vite with PWA plugin
- **Package Manager:** Bun

## Getting Started

### Prerequisites

- [Bun](https://bun.sh/) (or Node.js 18+)
- A [Convex](https://convex.dev/) deployment (free tier works)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/aeare.git
   cd aeare
   ```

2. **Install dependencies**
   ```bash
   bun install
   ```

3. **Start the Convex dev backend**
    ```bash
	bun run convex:dev
    ```
    This uses a hosted Convex dev deployment with live code sync. AeAre does not run a fully self-hosted local Convex backend in this repo.

4. **Configure environment variables**

	Create a `.env.local` file for the web app from `.env.example`:

	```bash
	# Convex URL used by the frontend client at runtime
	PUBLIC_CONVEX_URL=https://jovial-wildcat-461.convex.cloud

	# Optional fallback for Vite-only local workflows
	VITE_CONVEX_URL=https://jovial-wildcat-461.convex.cloud

	# Matching Convex auth issuer for the same non-production deployment
	PUBLIC_CONVEX_SITE_URL=https://jovial-wildcat-461.convex.site
	```

   Configure Convex Auth variables:

   ```bash
   # Local app URL used in magic links (must match your dev server protocol)
   bunx convex env set SITE_URL http://localhost:5173

   # Required by Convex Auth (usually set by `bunx @convex-dev/auth`)
   bunx convex env set JWT_PRIVATE_KEY "<private-key>"
   bunx convex env set JWKS '<jwks-json>'

   # Email provider key for magic links (Resend)
   bunx convex env set AUTH_RESEND_KEY <your-resend-key>
   ```

5. **Start the development server**
   ```bash
   bun dev
   ```

   Optional HTTPS mode (for devices/workflows that require TLS):

   ```bash
   DEV_HTTPS=true bun dev
   bunx convex env set SITE_URL https://localhost:5173
   ```

6. **Access the app**
   Open http://localhost:5173 in your browser. If you started with `DEV_HTTPS=true`, use https://localhost:5173 instead. Always keep `SITE_URL` on the same protocol/host as the running app to avoid redirect failures.

## Auth setup and recovery runbook

Phase 1 auth uses Convex Auth with magic links (Resend provider). Use this checklist when onboarding a new machine or recovering from auth issues.

### Required environment variables

Frontend (`.env.local` or `.env`):

```bash
PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
# optional local fallback
VITE_CONVEX_URL=https://your-deployment.convex.cloud
```

Convex deployment variables:

```bash
bunx convex env set SITE_URL http://localhost:5173

```

## Cloudflare deploy contract

Cloudflare build hooks are the only supported deploy path for AeAre previews and production. Local deploy commands are intentionally disabled in `package.json`.

### Fast local development loop

Use the fast Vite loop while iterating on features:

```bash
bun run dev
```

This is the quickest feedback loop, but it is not the final deploy gate.

### Built-worker parity loop

Use the parity loop whenever you need the Cloudflare-shaped runtime locally. These commands must point only at non-production Convex deployments, auth origins, and Cloudflare bindings.

```bash
bun run test:e2e:parity
bun run verify:deploy
```

- `bun run test:e2e:parity` builds the app and runs the parity-tagged Playwright smoke suite against Wrangler-served `.svelte-kit/cloudflare/_worker.js`.
- The parity runtime is served with `wrangler dev --local`, so it exercises the built worker locally without Cloudflare's remote Browser Rendering proxy.
- `bun run verify:deploy` is the local pre-push gate. It runs `bun run check`, `bun run build`, and `bun run test:e2e:parity` in order.

Parity smoke coverage currently verifies:

- Signed-out redirects still protect `/scan`, `/book/new`, and `/book/test-book-id`
- The AR API returns a graceful degradation response when Cloudflare Browser Rendering is unavailable in non-production parity work

### Preview builds

Pull request branches are deployed by Cloudflare to preview URLs in the form:

```text
https://<branch-name>-aeare.timothyclrk66.workers.dev/
```

Cloudflare preview config must point the app at the non-production Convex deployment:

- `PUBLIC_CONVEX_URL=https://jovial-wildcat-461.convex.cloud`
- `PUBLIC_CONVEX_SITE_URL=https://jovial-wildcat-461.convex.site`

The matching Convex deployment should keep preview-safe auth and email settings only.

### Production build

Merged production code is deployed by Cloudflare to:

```text
https://aeare.thedailyclark.com/
```

Cloudflare production config must point the app at the production Convex deployment:

- `PUBLIC_CONVEX_URL=https://confident-dove-172.convex.cloud`
- `PUBLIC_CONVEX_SITE_URL` must point at the matching production Convex auth issuer for that deployment

Convex production secrets and auth settings should live only in the production Convex deployment and Cloudflare dashboard.

### Rollback and smoke checks

- Roll back by redeploying the last known-good Cloudflare build plus the last known-good Convex deployment for that environment.
- After a Cloudflare preview or production deploy, smoke-check the signed-out redirects for `/scan`, `/book/new`, and `/book/test-book-id`.
- Smoke-check the AR endpoint and confirm it either returns AR data or the documented graceful fallback message instead of crashing when Browser Rendering is unavailable.
- If auth redirects or AR fallback differ from parity results, stop and fix parity before promoting the next build.

### Required secrets, bindings, and variable sources

Copy only from the safe templates and service dashboards; never commit real values.

App/runtime template (`.env.example`) for local development:

- `PUBLIC_CONVEX_URL` — non-production Convex deployment URL for local development and local parity
- `VITE_CONVEX_URL` — optional local fallback for Vite-only workflows
- `PUBLIC_CONVEX_SITE_URL` — non-production Convex site URL for local auth/parity
- `CONVEX_DEPLOY_KEY` — local dev/test helper only; keep secret and out of git

Worker template (`.dev.vars.example`) for local parity only:

- `CONVEX_SITE_URL` — Convex auth issuer URL for the local non-production deployment
- `SITE_URL` — local app origin used for auth callbacks
- `AUTH_RESEND_KEY` — Resend API key from the Resend dashboard
- `AUTH_EMAIL_FROM` — verified sender identity for magic-link delivery
- `JWT_PRIVATE_KEY` — Convex Auth signing key
- `JWKS` — Convex Auth JWKS payload
- `AEARE_AUTH_EMAIL_MODE` — local inbox override for automated non-production testing only

Cloudflare dashboard/runtime config:

- Preview branch builds must keep Browser Rendering and all vars/bindings pointed at non-production resources
- Production must not reuse preview/local test overrides

Production should not keep the local inbox overrides:

```bash
AEARE_AUTH_EMAIL_MODE
AUTH_EMAIL_MODE
```

Unset those in production if they exist.

### Provider setup checklist (Resend or local dev inbox)

1. Create a Resend account and API key.
2. Verify your sending domain/address in Resend.
3. Set `AUTH_RESEND_KEY` in Convex env.
4. Confirm `SITE_URL` matches your app origin.
   - If running `DEV_HTTPS=true bun dev`, `SITE_URL` must be `https://localhost:5173`.
5. In the app, use **Sign in** and enter an email to send a magic link.

For local automated testing, switch the Convex deployment to the built-in dev inbox delivery mode:

```bash
bunx convex env set AEARE_AUTH_EMAIL_MODE local
```

Switch back to Resend delivery with:

```bash
bunx convex env unset AEARE_AUTH_EMAIL_MODE
```

### Local development notes

- Run `bun run convex:dev` in one terminal and `bun run dev` in another.
- `bunx convex dev` uses a hosted development deployment, not a fully local database/runtime.
- For local auth automation, keep `AEARE_AUTH_EMAIL_MODE=local` on the dev deployment and use the built-in dev inbox route.

### Automated UI checks

Run the signed-out route-protection checks with:

```bash
bun run test:e2e
```

Current automation covers:

- Public home remains usable while signed out
- Signed-out redirects from `/scan`, `/book/new`, and `/book/[id]`
- Full local magic-link sign-in when `AEARE_AUTH_EMAIL_MODE=local`

Important note:

- The local inbox is stored in the Convex dev deployment instead of Mailpit. This keeps the same Convex Auth magic-link flow while avoiding the network reachability problem of a hosted Convex action trying to deliver mail to your laptop's `localhost`.

### Recovery guide

- **Magic link email not sent**
  - Confirm `AUTH_RESEND_KEY` is set in Convex env.
  - Confirm sender domain/address is verified in Resend.
  - Check Convex function logs for provider errors.

- **Signed in, but account bootstrap fails**
  - Confirm `JWT_PRIVATE_KEY`, `JWKS`, and `SITE_URL` are set in Convex env.
  - Confirm magic link is opened in the same app environment that requested it.
  - Retry using the in-app **Retry authentication** button.

- **401/403 from Convex after opening magic link**
  - Sign out and sign in again to refresh local auth tokens.
  - Re-run `bunx convex env list` and verify auth variables.
  - Ensure `src/convex/auth.config.ts` still points at `CONVEX_SITE_URL` + `convex` audience.

- **Stuck callback (`?code=...`)**
  - Ensure `SITE_URL` points at the served app origin.
  - Clear site storage for local host and retry sign-in.

## Usage

### Adding a Book

1. Tap the scan button (+) on the home screen
2. Allow camera access when prompted
3. Point your camera at a book barcode
4. The app will automatically look up the book and populate the form
5. Confirm or edit the details and save

### Tracking Reading Progress

1. Open any book in your library
2. Mark the book as "read" using the toggle
3. Select the date the book was finished
4. Optionally enter an AR quiz score

### Offline Use

The app is designed to keep key flows usable with degraded behavior when network or AR lookups fail. True offline persistence remains an active verification item.
